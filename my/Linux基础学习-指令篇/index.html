Linux基础学习-指令篇

发表于 2019-07-11 | 更新于 2019-07-15 | 分类于 Linux

Linux指令学习

上一篇讲解了权限，这一篇用来讲常见的指令。
该文章从一开始进入Linux后该如何操作，以及操作指令带来什么作用。

前言

在Linux的世界中，指令是不可避免的，虽然有的Linux发行版也是有界面的，但是服务器基本不会有，
因为画面意味着需要更多的带宽，意味着画面延迟！这是我们不想见到的，所以指令学习不可避免。

基础指令

当我们刚进入Linux界面的时候，我们首先得知道我们在哪里？然后才能去寻找我们想要的东西，
接着才能查看内容，以及修改内容。

pwd

pwd指令显示用户当前目录。用来查看当前在哪个文件夹下。
用法：pwd

ls

ls指令查看当前目录下所有的文件/文件夹的名称。
用法：ls 路径(直接调用ls是查看当前文件夹下的文件)

  • ls(哦！这时候就得提一下每个指令或多或少都有相对应参数！参数的不同，导致输出的结果打不一致哦~当然指令之间也能组合如ls -lh)
    • -l：列表形式列出详细信息
    • -a：显示所有文件(包含.开头的隐藏文件/文件夹)
    • -h:显示文档大小的时候以可读性较高的形式显示，一般搭配l指令，

cd

cd指令用于文件之间的切换，俗话说进入某文件夹。
用法：cd 路径

  • cd(用法展示cd ..，下面不是参数哦。)
    • ~：进入用户家目录(每个用户的家目录一般都是不同的。)
    • ..：返回上级目录
    • -：返回最近访问目录
    • /:返回根目录(根目录唯一)。

cat、tac

cat指令用于查看文件内容。
tac功能同上！但是是从文件最后查看。(仔细的人会发现tac就是cat的倒写，哈哈就是这么有趣)
用法：cat 文件的路径(可以直接是文件名)

  • cat
    • -n：输出文本有行号。
    • -b：对空白行不占用行号。
    • -s：对两行以上空行，自动默认成一行。

除了查看文件内容，还有一个用途：对文件进行合并。
用法：cat 文件路径 文件路径 文件路径>合并后路径
(文件路径有空格隔开。>大于符号重定向输出后面在讲解)

mkdir

mkdir指令用于创建文件夹。
用法：mkdir 文件路径(路径可以直接是文件名)

  • mkdir
    • -p：用于创建多层级文件夹。(不用该指令就只能创建好了进入创建的目录在创建，否则报错)

rmdir

rmdir指令用于删除文件夹。
用法：rmdir 文件路径

touch

touch指令用于修改文件或者目录的时间属性，如文件不存在则创建文件。
用法：touch 文件路径(路径可以直接是文件名)

  • touch(一般也就拿来创建文件了。)
    • -c：不创建任何文件
    • -a：更改文件存取时间
    • -m：更改文件修改时间

cp

cp(copy)复制文件
用法：cp 被复制的文档路径 文档被复制到的路径

  • cp
    • -a：相当于-dpR
    • -d：复制时保留链接(就是快捷方式)
    • -p：除复制文件的内容外，还把修改时间和访问权限也复制到新文件中。(常用于备份)
    • -r：如果复制的是文件夹，则复制该文件夹所有内容。

mv

mv(move)移动，剪切
用法：mv 需要移动的文档路径 需要保存的位置路径

  • mv
    • -f：不询问直接覆盖。
    • -i：询问后覆盖。

还有一个修改文件名用途，用法mv 目录名 目录名(或者是文件名 文件名)

rm

rm(remove)移除，删除
用法：rm 参数 需要移除的文档路径

  • rm(该命令慎重使用。)
    • -f：强制删除。
    • -r：目录及以下全部删除。

find

find用于查找文件(参数非常多)
用法：find 搜索路径 选项 选项值

  • find
    • name：按文件名搜索(支持模糊查询)
    • type：按照类型(权限篇里讲过类型如：-表示文件d表示文件夹)

输出重定向

>、>>该指令是输出重定向。
<、<<该指令是输入重定向。

一般命令的输出都会显示在终端中，有些时候需要将一些命令的执行结果想要保存到文件中进行后续的分析/统计，则这时候需要使用到的输出重定向技术。

>：**覆盖**输出，会覆盖掉原先的文件内容
>>：**追加**输出，不会覆盖原始文件内容，会在原始内容末尾继续添加

语法：正常执行的指令 > / >> 文件的路径
注意：文件可以不存在，不存在则新建。
简单举例：当我们使用ls的时候一般屏幕会直接打印出我们查看的当前目录列表，但是当指令变成ls > ls.txt时候他不会吧指令打印出来而是保存在我所写的文件里面(注意一个是覆盖！两个是追加)。

进阶指令

df

df指令用于查看硬盘空间。
用法：df -h

  • df
    • -h 以可读性较高的形式展现

free

free指令用于查看内存使用情况。
用法：free -m

  • free
    • m以mb单位查看

head

head指令查看文件的前n行(默认n为10)。
用法：head -n 文件路径

tail

tail指令用于查看文件的倒数n行(默认n为10)。
用法：tail -n 文件路径

less

less指令用于查看文件，以较少的内容输出。(cat是全部输出)
该指令使用时，最下端有：可以输入对应指令进行操作。
用法：less 需要查看的文件路径

  • 空白键、pagedown：下一页
  • b、pageup：上一页
  • /字符(斜杠也是指令)：向下搜索字符
  • ?字符：向上搜索字符
  • g：跳到文件开头
  • G：跳到文件结尾
  • q：退出查看

more

more指令也是用于查看文件，但是能查看文档当前百分比。(看到百分百后自动退出)

  • 空白键：下一页
  • b：上一页
  • q：退出
  • /字符：向下搜索字符

which

which在环境变量$PATH设置的目录里查找符合条件的文件。
用法：which 文件名 一般用于查询指令的路径如：which ls

whereis

whereis也用于查询文件(主要针对 /bin /sbin和/usr/share/man)

wc

wc指令用于统计内容。
用法wc 指令 文件路径/文件名

  • wc
    • l：表示lines，行数
    • w：表示words，单词数( 根据空格判断)
    • c：表示bytes，字节数

grep

grep指令用于过滤，通常配合管道使用。

管道(很牛且常用)

|该指令为管道符一般可以用于过滤，”特殊”，”扩展处理”。。
用法：cat test.txt | wc -w，查看test有多少个字符。

cat test.txt | wc -w管道符符号隔开，可以看两部分。
前面的cat test.txt输出结果，是wc -w的输入。
再来一个例子：ls / | grep y
查看/(根目录)下名字有y的文档。步骤先查看ls然后ls的结果在进行过滤包含y的结果。

扩展：查看跟(/)目录下有多少个文档个数。
ls / | wc -l

清屏

当屏幕一大堆无用看着不舒服，ctrl+L一下世界就清净了。

总结

linux的指令基本就够了~增删改查，进文件进目录等等，只不过以上指令学会了，也就等同于有个鼠标操作win系统罢了，后面我还会记录一些高级指令，比如查看进程，杀死进程，启动服务，安装软件等…
