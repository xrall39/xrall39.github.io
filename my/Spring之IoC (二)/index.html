Spring之IoC (二)

发表于 2019-04-05 | 更新于 2019-04-08 | 分类于 Java

　　之前关于IoC（控制反转）这个思想写了一大堆总结（废话），也算是对其有了一定的了解，那么之前写这么多理论的东西，这一篇就该写一下Spring使用方式。
　　首先，关于Spring的IoC我们需要了解以下几点：

关于Bean

IoC容器
　　IoC容器就是具有依赖注入功能的容器，IoC容器负责实例化、定位、配置应用程序中的对象及建立这些对象间的依赖。最直白的展示就是能代替我们new且代替了我们手动传参注入。

Bean概念

　　 由IoC容器管理的那些组成你应用程序的对象我们就叫它Bean。Bean就是由Spring容器初始化、装配及管理的对象，除此之外，bean就与应用程序中的其他对象没有什么区别了。

  • Bean行为定义：这些定义了Bean在容器中的；包括作用域（单例、原型创建）、是否惰性初始化及生命周期等；
  • Bean创建方式定义：说明是通过构造器还是工厂方法创建Bean；
  • Bean之间关系定义：即对其他bean的引用，也就是依赖关系定义，这些引用bean也可以称之为同事bean 或依赖bean，也就是依赖注入(DI)。

　　Spring里容器是通过XML配置文件来进行管理Bean的所以先从配置文件入手。直接配置文件先展示一段对比看一下。

<beans xmlns="http://www.springframework.org/schema/beans"
       xmlns:xsi="http://www.w3.org/2001/XMLSchema-instance"
       xsi:schemaLocation="http://www.springframework.org/schema/beans
       http://www.springframework.org/schema/beans/spring-beans.xsd"> 这是一堆基本信息

<bean id="userService" class"FQN" scope="singleton">
// UserService userservice = new UserServiceImp(); 这一句代码等同于上面一段标签体

</beans>

　　FQN： 全限定类名。就是项目中你所创建该类的位置如：com.soecode.xr.service
　　
　　bean标签：

  • id：标识符或名字（唯一）。
  • name：同样为标识符（唯一）。
  • class：指定类位置也就是用FQN表示。
  • scope：英译为作用域，我们用来定义单例还是多例。

singleton（单例）：只有一个共享的实例存在，所有对这个bean的请求都会返回这个唯一的实例。
prototype（多例）：对这个bean的每次请求都会创建一个新的bean实例，类似于new。

实例化Bean

第一种：使用构造器实例化Bean

　　如果是默认空构造函数一句话搞定

<bean id="bean1" class"FQN">

　　但是有可能你实例化的时候他还要带参数对吧，那么就需要在配置详细点

<bean id="bean2" class="FQN">  
<!-- 指定构造器参数 -->  
     <constructor-arg index="0" value="Hello Spring!"/>  
//Public userService(String message) Hello Spring就传给 userService
</bean>

　　index表示位置，value表示常量值，也可以指定引用（可能传入的参数是一个类所以要引用另外一个 bean）
　　其实这里应该有疑问，既然Spring代替我们手动new那我们又如何使用呢？

第二种：静态工厂实例化Bean

先写一个静态工厂

静态工厂写完了，接着写 XML 配置文件 

<bean id="bean3" class="FQN" factory-method="newinstance">
    <!-- 指定构造器参数 -->  
     <constructor-arg index="0" value="Hello Spring!"/>  
</bean>

应该一眼就看出来多了一个 factory-method （工厂方法）这个里面的内容其实就是上面定义的BeanStaticFactory里面的方法名，如果BeanStaticFactory静态工厂里面有许多工厂方法，你不指定谁知道生成哪一个。对吧？
最后调用也是一样的

第三种：实例工厂实例化Bean

　　　　这一种和上面非常相识 只是工厂不是静态(static)的而已。

接着还是写XML 但是这个和上面的静态是有区别

<!— 1、定义实例工厂Bean -->  
<bean id="beanInstanceFactory" class="FQN" />
<!- 2、实例化工厂 -->
<bean id="bean4" class="FQN" factory-bean="beanInstanceFactory" factory-method="newInstance">
    <constructor-arg index="0" value="Hello Spring!"/>
</bean>

一看就发现实例工厂相对于静态有两个不同一个是他需要 自己先申明一次bean 然后在factory-bean指定，
最后使用方式都是一样的

三种方法就算是总结完了，如果看完后回忆一下就会发现最后我们的获取方式几乎都是一模一样的。但是配置文件却是不同的，Spring IoC帮你创建Bean，我们只管使用就可以了，这就是Spring的魅力所在。

总结

　　这两篇文章，写完基本上IoC的基础算是完事了，到现在我们只涉及到了Bean的实例化，还没有涉及到Bean之间的关系，后面再写吧。而且关于Bean的命名我也没有总结。
