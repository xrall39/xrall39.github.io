Java中Entity、VO、DTO区别

发表于 2019-05-04 | 分类于 Java

Entity、VO、DTO这些词语一开始是接触到当时也没怎么在意，反正就是来装数据的嘛~，但是随着后面学习看别人的项目渐渐的发现别人都会分出来，而自己就好像一招鲜吃遍天只会Entity到底。随着时间的推移，这个问题就一直在脑海中浮现，这几个都是装数据有啥区别啊？到底怎么个用法啊？什么时候用Entity、什么时候VO，什么时候DTO呢?那先给每个词给个解释吧。

Entity：通常而言我们就是来对应数据库字段的。
VO：view Object，一看名字就知道是 和前端页面交互用的类
DTO：Data Transfer Object，数据传输对象，数据在什么时候传输？那肯定就是前端到进数据库这一段路对吧。

上面大概就是这么个意思吧，尽管如此，VO和DTO在大部分情况下是一致的，所以导致了感觉大多数都是在用DTO或者VO直接用来传输，这时候就会有一点疑惑，既然都差不多那还分个DOT和VO干嘛，其实导致的原因是： 这是一种设计模式上的思维，所以才有了这两个感觉上一样又不一样的东西，但是两者却有本质的区别，DTO代表服务层需要接收的数据和返回的数据，而VO代表展示层需要显示的数据。Entity是最容易理解的，就是数据库里什么字段，里面就写一个什么属性。
以上解释感觉差不多了，在来一个网上例子滚固一下最好。

VO出场

这个想法是没错的，做法也的确没错，只不过唯一的是你得知道有这么一个东西，这是一种概念，分层次的概念，你可以不用但是得知道。
